# EMR 분석 및 UI/UX 표준 가이드 (EMR Analysis Standard)

본 문서는 바른 컨설팅의 EMR 데이터 분석 대시보드 구축 시 적용되는 표준 설계 지침입니다. 모든 EMR 시스템(오케이차트, 한차트 등)은 아래의 표준을 준수하여 일관된 사용자 경험을 제공해야 합니다.

## 1. UI/UX 레이아웃 표준 (Layout Standard)

### 1.1. 월별 네비게이션 (Rolling 12-Month Navigation)
- **가시성**: 스크롤 없이 한 화면에 최근 12개월이 모두 표시되어야 함.
- **연도 관리**: 연도별 탭(Tab)을 제공하여 과거 데이터 접근성을 보장함.
- **상태 표시**: 
  - `선택된 월`: 메인 컬러(Gold)로 강조.
  - `비교 대상 월`: 보조 컬러(Blue)로 구분.
  - `데이터 없음`: 비활성화(Dimmed) 처리.

### 1.2. 데이터 관리 (Header Control)
- **중앙 집중화**: 데이터 업로드(Upload)와 초기화(Trash) 버튼은 페이지 상단 우측 헤더로 일원화.
- **시인성**: 업로드 버튼은 강조색(#FBBF24)과 애니메이션을 적용하여 사용자의 행동 유도.

## 2. 분석 및 데이터 로직 표준 (Analysis Logic Standard)

### 2.1. 자동 비교 시스템 (Auto-Comparison)
- **MoM (Month-over-Month)**: 특정 월 클릭 시, 별도의 설정 없이 직전 월과 자동으로 비교 분석 수행.
- **YoY (Year-over-Year)**: '작년 대비' 버튼을 통해 1년 전 동월 데이터와 즉시 비교 기능 제공.

### 2.2. 핵심 지표 체계 (Key Metrics)
모든 EMR 대시보드는 아래 4대 수익 지표를 반드시 포함해야 함:
1. **보험 수익**: 본인부담금 + 보험청구액 합계.
2. **비급여 수익**: 비급여 진료비 총액 및 전체 매출 대비 비중(%).
3. **자동차보험(자보)**: 자보 청구액 및 내원 환자 수.
4. **산재보험**: 산재 청구액 정보.

### 2.3. 환자 통계 바 (Patient Statistics Bar)
- 총 내원 환자, 신규 환자, 자보 환자, 일평균 환자 수를 하단 바 형태로 상시 노출.
- 모든 지표에는 지난달 대비 증감 수치(▲/▼)를 병기함.

---

## 3. 시각화 표준 (Visualization Standard)

### 3.1. 트렌드 차트 (Interactive Trend Chart)
- **최소 높이 보장**: 데이터가 적은 달도 차트 막대가 사라지지 않도록 최소 높이(예: 8%) 설정.
- **인터랙션**: 차트의 막대 클릭 시 해당 월의 상세 리포트로 즉시 전환.
- **복합 차트**: 매출(막대)과 환자 수(라인/포인트)를 결합하여 상관관계 파악 용이성 증대.

### 3.2. 디자인 에스테틱 (Premium Design)
- **Theme**: Deep Dark Mode (#0A0E1A 기반).
- **Typography**: 수치 데이터는 Black(900) 웨이트와 Tighter 자간을 사용하여 가독성과 전문성 강조.
- **Animation**: 숫자 롤링(Rolling Number), 그라데이션 애니메이션 등을 통해 생동감 있는 인터페이스 구현.

---

## 4. 데이터 파싱 표준 (Parsing Standard)

- **포맷**: 모든 날짜는 `YYYY-MM` 형식을 표준으로 사용.
- **정밀도**: 원단위 절사 및 일평균 환자 수의 소수점 첫째 자리 유지.
- **확장성**: `excelParser.ts`를 통해 EMR별 고유 엑셀 양식을 표준 데이터 객체로 변환하는 레이어 유지.
